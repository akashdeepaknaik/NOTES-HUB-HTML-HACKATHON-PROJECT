Lecture 1 — How DNS Works?

A kid-friendly explanation with a gaming analogy.

Short Summary

DNS (Domain Name System) is the internet's phonebook. It turns names like google.com into IP addresses like 142.250.72.206. Your browser then uses that address to connect to the correct server.


Gaming-World Analogy
Analogy: Finding a friend's house like DNS

  1. You (Browser) check memory: Do I already remember DragonSlayer99's house?


  2. Check diary (OS cache): Look into your notes.


  3. Ask gatekeeper (Router): Maybe he remembers.


  4. Call Local Guide (Resolver): If not, he promises to find it.


  5. City Office (Root Servers): Guide asks which area handles ".com".


  6. Area Office (TLD): Guide asks who manages "google.com".


  7. Google’s Office (Authoritative): Exact house number (IP) is given.


  8. Guide returns → Gives you the IP and remembers it for later.


  9. You connect: You now enter the correct house (website/game).

DNS Flow (Technical)
How DNS Resolves Step by Step
  1. Browser checks browser cache.


  2. If missing → OS cache.


  3. If missing → Router cache.


  4. If still missing → Recursive Resolver (ISP/public).


  5. Resolver asks Root → TLD → Authoritative servers.


  6. Authoritative server returns IP.


  7. Resolver caches the IP and sends it back.


  8. Browser connects via IP (usually HTTPS, port 443).

Real Example
DNS in Action: Google.com


If you type Google.com:

  1. DNS resolves it to 142.250.72.206
  2. Browser then loads Google’s homepage from that IP.



Quick Explanation:

  1. Browser checks its cache → OS cache → Router cache.
  2. If not found, query goes to Resolver (ISP/public).
  3. Resolver asks Root → TLD → Authoritative server.
  4. Authoritative server gives final IP.
  5. Browser connects to the server using that IP.

DNS Cheat Answer
One Line DNS Definition

DNS is the internet's phonebook: it maps human-readable names (Google.com) to IP addresses (142.250.72.206). The browser uses this IP to connect to the server.

It converts easy-to-remember domain names like Google.com into machine-readable IP addresses like 142.250.72.206


  • Without DNS, we would need to memorize long strings of numbers for every website.

  • DNS works silently in the background, completing the lookup in milliseconds.

  • Once resolved, the IP is cached so the next visit is even faster.

  • It ensures that no matter where you are in the world, typing Google.com takes you to the correct Google server.

In short: DNS makes the web simple, fast, and human-friendly by bridging the gap between names and numbers.


Glossary
DNS
Domain Name System – translates domain names to IP addresses.
IP Address
Unique number that identifies a server on the internet.
Resolver
Server that finds the IP address for your query.

Quick FAQ
Q: What if DNS fails?

A: You see an error like "DNS_PROBE_FINISHED_NXDOMAIN".

Q: Who provides DNS?

A: Usually your ISP, or public resolvers like Google (8.8.8.8).

Q: Is DNS secure?

A: DNS can be spoofed, that’s why DNSSEC was introduced.


Useful Commands 
Windows:
  nslookup google.com

macOS / Linux:
  dig google.com
       or
  nslookup google.com

These commands show the IP address for a domain.
