File Path System

Fact: A website is a collection of files (HTML, CSS, JS, Images).



  • Same Folder → <a href="about.html">About</a>



  • Sub Folder → <img src="images/logo.png">



  • Parent Folder → <img src="../images/logo.png">



  • Absolute URL → <img src="https://example.com/logo.png">

HTML Boilerplate

Every HTML document starts with this boilerplate structure.

 Code 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

Multipage Websites

A website usually has many pages (Home, About, Contact). All share the same header & footer.

 Code 
<nav>
  <a href="home.html">Home</a> |
  <a href="about.html">About</a> |
  <a href="contact.html">Contact</a>
</nav>

div, class, id
  • id → Unique identifier for one element.



  • class → Reusable for multiple elements.
 Code 
<div id="unique">Unique</div>
<div class="card">Card 1</div>
<div class="card">Card 2</div>
<div id="special" class="card">Special</div>

Quick FAQ
Q: Relative vs Absolute Path?

A: Relative = depends on folder structure, Absolute = complete URL.

Q: Why use semantic tags?

A: For readability, SEO, and accessibility.
