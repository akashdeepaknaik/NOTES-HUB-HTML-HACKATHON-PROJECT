Lecture 6 — HTML Media

Topics: Video, Audio, Video Attributes, Cross-Browser Media, Accessibility, Copyright

Table of Contents
  • First Principle
  • Video Attribute
  • Cross Browser Media
  • Accessibility with Tracks
  • Copyright Symbel

First Principle

The web is more than just text and images. Earlier browsers needed plugins like Flash or QuickTime to play media, which was inefficient and insecure.

Solution: HTML5 introduced <video> and <audio> tags for native, plugin-free media.

 Example:
<video src="my-awesome-video.mp4"></video>
<audio src="audio/theme-song.mp3" controls></audio>

Video Attributes
  • width / height → Set player size
  • autoplay + muted → Start video automatically (with no sound)
  • loop → Restart video automatically
  • poster → Thumbnail image before play
 Example:
<video src="video.mp4" width="640" height="360" autoplay muted loop poster="thumbnail.jpg" controls></video>

Cross-Browser Media

Different browsers support different formats. Use <source> inside <video> for better compatibility.

 Example:
<video controls poster="images/thumbnail.jpg">
  <source src="videos/my-video.webm" type="video/webm">
  <source src="videos/my-video.mp4" type="video/mp4">
  Sorry, your browser doesn't support embedded videos.
</video>

Accessibility with <track>

Add captions or subtitles to make your media accessible to all users.

 Example:
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track src="captions_en.vtt" kind="captions" srclang="en" label="English">
  <track src="subtitles_es.vtt" kind="subtitles" srclang="es" label="Español">
</video>

© Copyright Symbol

Use &copy; or &#169; to add copyright symbol in footer.

 Example:
<footer>
  <p>Copyright &copy; 2024 My Awesome Website. All Rights Reserved.</p>
</footer>
